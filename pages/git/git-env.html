Настройка vscode

Global vscode settings

                        
{
    "editor.defaultFormatter": "esbenp.prettier-vscode",
    "[svg]": { "editor.defaultFormatter": "jock.svg" },
    "[markdown]": { "editor.defaultFormatter": "DavidAnson.vscode-markdownlint" },

    // Вывод всех настроек в левой части экрана
    "workbench.settings.useSplitJSON": true,
    "workbench.colorTheme": "Night Owl (No Italics)",
    "workbench.iconTheme": "icons",

    "files.insertFinalNewline": true,
    "files.trimFinalNewlines": true,
    "files.trimTrailingWhitespace": true,

    "editor.fontFamily": "'Dank Mono', Consolas, 'JetBrains Mono', Inconsolata, monospace",
    "editor.wordWrap": "on",
    "editor.mouseWheelZoom": true,
    "editor.snippetSuggestions": "top",
    "editor.renderWhitespace": "none",
    "explorer.compactFolders": false,
    "breadcrumbs.enabled": true,

    // настройка на уровне проекта, показывает в какой части работать относительно корня
    // "terminal.integrated.cwd": "html"

    "terminal.integrated.profiles.windows": {
        "Git Bash": { "path": "C:\\Program Files\\Git\\bin\\bash.exe" }
    },
    "terminal.integrated.defaultProfile.windows": "Git Bash",

    "javascript.suggest.names": false,

    // EXT Colorize
    "colorize.languages": ["css", "scss", "javascript", "sass"],
    "colorize.colorized_variables": ["css", "scss", "javascript"],
    "colorize.hide_current_line_decorations": false
}

Дальнейшая установка

Что делаем

  • установка wsl2 на другой диск с помощью видео номер 3

  • изменить json файл терминала

  • Установить расширение Remote Development

  • sudo apt update && sudo apt upgrade

  • командой 'code .' через консоль подключаем wsl2 к vscode

  • Git setup

  • nvm/nodejs/npm

Дополнительно

  • Добавить в рабочую WSL2 директорию ссылки на диски Windows (что-бы не вводить каждый раз /mnt/c/...)

    • cd ~

    • ln -s /mnt/c/Users/user win_home
