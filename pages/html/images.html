Performance

Use fetchpriority and loading="lazy" for better loading performance. 

                        
<img
src="hero.jpg"
alt="A description of the image."
width="1200"
height="800"
loading="eager"
>

<img
 src="hero.jpg"
 alt="A description of the image."
 width="1200"
 height="800"
 loading="eager"
 fetchpriority="high"
>

loading eager - default

decoding: async or sync.

Use the sync value if the image itself is the most important piece of content to prioritize. 

                        
<img
  src="image.png"
  alt="A description of the image."
  width="300"
  height="200"
  loading="lazy"
  decoding="async"
>

<img
  src="hero.jpg"
  alt="A description of the image."
  width="1200"
  height="800"
  loading="eager"
  decoding="sync"
>

srcset

                    
<img
  src="small-image.png"
  alt="A description of the image."
  width="300"
  height="200"
  loading="lazy"
  decoding="async"
  srcset="small-image.png 300w,
   medium-image.png 600w,
   large-image.png 1200w"
>

Always use sizes with srcset

                    
<img
  src="small-image.png"
  alt="A description of the image."
  width="300"
  height="200"
  loading="lazy"
  decoding="async"
  srcset="small-image.png 300w,
   medium-image.png 600w,
   large-image.png 1200w"
  sizes="(min-width: 66em) 33vw,
   (min-width: 44em) 50vw,
   100vw"
>

Pixel density descriptor

                    
<img
  src="small-image.png"
  alt="A description of the image."
  width="300"
  height="200"
  loading="lazy"
  decoding="async"
  srcset="small-image.png 1x,
   medium-image.png 2x,
   large-image.png 3x"
>

Picture element

                        
<picture>
  <source srcset="full.jpg" media="(min-width: 75em)" width="1200" height="500">
  <source srcset="regular.jpg" media="(min-width: 50em)" width="800" height="400">
  <img src="cropped.jpg" alt="A description of the image." width="400" height="400" loading="eager" decoding="sync">
</picture>

Multiple image formats

                        
<picture>
    <source srcset="image.avif 1x, image.avif 2x" type="image/avif" media="(max-width: 575px)" />
    <source srcset="image.webp 1x, image.webp 2x" type="image/webp" /> | <source srcset="image.webp 200w, image.webp 200w" type="image/webp" />
    <img
        src="image.jpg"
        srcset="
            image.jpg 1x,
            image.jpg 2x
        "
        alt="image"
    />
</picture>

Same with background-image

                        
background-image: image-set(
    url("img/cat@2x.webp") type("image/webp") 2x,
    url("img/cat@1x.webp") type("image/webp") 1x,
    url("img/cat@2x.avif") type("image/avif") 2x,
    url("img/cat@1x.avif") type("image/avif") 1x,
    url("img/cat@2x.jpeg") 2x,
    url("img/cat@1x.jpeg") 1x
);

From GIF, JPG, PNG and AVIF - better use AVIF. With the same look, it`s 30% smaller.

Always use max-width: 100%; and height: auto; on images, width dimension in html. Browser use html dimensions for calculating aspect-ration of images, for better performance.

Use custom hotkey in vs code, emmet - update image size .

Squoosh API

Not maintainable

Need install:

  • @squoosh/cli globally

  • Change node to 16

                        
squoosh-cli --mozjpeg '{quality: 75}' -d img/jpg *(*.jpg|*.png|*.avif|*.webp!(webp|avif|jpg))
squoosh-cli --avif '{quality: 100}' -d img/avif *(*.jpg|*.png|*.avif|*.webp!(webp|avif|jpg))

squoosh-cli --webp '{ quality: 90, lossless: 0, effort: 6 }' -d img/webp *(*.jpg|*.png|*.avif|*.webp!(webp|avif|jpg))

squoosh-cli --mozjpeg '{quality: 75}' -d jpg *(*.jpg|*.png|*.avif|*.webp!(webp|avif|jpg)) && squoosh-cli --avif '{quality: 100}' -d avif *(*.jpg|*.png|*.avif|*.webp!(webp|avif|jpg)) && squoosh-cli --webp '{ quality: 90, lossless: 0, effort: 6 }' -d webp *(*.jpg|*.png|*.avif|*.webp!(webp|avif|jpg))

Useful

Картинке сохранять пропорции при ресайзе

                        
img {
    background-size: cover;
    max-width: 500px;
    aspect-ratio: 1 / 1;
}

Ссылка на всю карточку

                        
parent > h2 > a

.parent {
    position: relative;
}

a::before {
    position: absolute;
    top: 0;
    right: 0;
    left: 0;
    bottom: 0;
    content: "";
}

Outline на всю картинку

a:focus-visible {
    outline: none;
}

a:focus-visible::before {
    outline: 5px solid deepskyblue;
    outline-offset: -5px;
}

aspect-ratio of the images will remain constant

                        
img,
video,
iframe {
    max-inline-size: 100%;
    block-size: auto;
}

different aspect-ratio

                        
img {
    max-inline-size: 100%;
    block-size: auto;
    aspect-ratio: 2/1;
}

object-fit on images

                        
img {
    max-inline-size: 100%;
    block-size: auto;
    aspect-ratio: 2/1;
    object-fit: contain / cover;
}

img {
    max-inline-size: 100%;
    block-size: auto;
    aspect-ratio: 2/1;
    object-fit: cover;
    object-position: top center;
}

preload images what used in js or background css

                        
<link rel="preload" href="hero.jpg" as="image" fetchpriority="high">
