HTML: Images

Images are very important to beautify as well as to depict many complex concepts in simple way on web page.

About

Use good images

  1. Different types of images: product photos, storytelling photos, illustrations, patterns

    • SVG PATTERNS
    • NICE ILLUSTRATIONS

  2. Use images to support website`s message and story . So only use relevant images!

Use images well

  1. Try to show real people to trigger user`s emotions

  2. If necessary, crop images to fit your message

  3. Experiment combining photos, illustrations and patterns

Handling text on images

  1. Darker or brighten image (completely or partially, using a gradient)

  2. Position a text into neutral image area

  3. Put text in a box

Some technical details

  1. To account for high-res screens, make image dimensions 2x as big as their displayed size

    • If image have website size 300x300, he`s original size need to be 600x600
    • Scale factor: Actual pixels the screen contains / Pixels represented on screen
    • On high-res screens, scale factor is 2x or even 3x, on "normal" screens it`s just 1x (1 physical pixel = 1 design pixel)

  2. Compress images for a lower file size and better performance

  3. When using multiple images side-by-side, make sure they have the exact same dimensions

  4. Always declare images size

Performance

  1. Use fetchpriority and loading="lazy" for better loading performance.

    • loading eager - default
                                    
<img
  src="hero.jpg"
  alt="A description of the image."
  width="1200"
  height="800"
  loading="eager" / loading="lazy"
  fetchpriority="high">

  2. decoding: async or sync.

    • Use the sync value if the image itself is the most important piece of content to prioritize. 
                                    
<img
  src="image.png"
  alt="A description of the image."
  width="300"
  height="200"
  loading="lazy"
  decoding="async" / decoding="sync">

    <source srcset="image.avif 1x, image.avif 2x" type="image/avif" media="(max-width: 575px)" />
    <source srcset="image.webp 1x, image.webp 2x" type="image/webp" /> | <source srcset="image.webp 200w, image.webp 200w" type="image/webp" />
    <img

            image.jpg 1x,
            image.jpg 2x
        "
        alt="image"
    />
</picture>

  4. srcset

                                    
<img
    loading="lazy"
    decoding="async"
    srcset="small-image.png 300w, medium-image.png 600w, large-image.png 1200w">

  5. srcset

                                    
<img
srcset="small-image.png 300w, medium-image.png 600w, large-image.png 1200w"
sizes="(min-width: 66em) 33vw, (min-width: 44em) 50vw, 100vw">

    Always use sizes with srcset

  6. Pixel density descriptor 

                                    
<img
srcset="small-image.png 1x, medium-image.png 2x, large-image.png 3x">

  7. Same with background-image 

                                    
background-image: image-set(
    url("img/cat@2x.webp") type("image/webp") 2x,
    url("img/cat@1x.webp") type("image/webp") 1x,
    url("img/cat@2x.avif") type("image/avif") 2x,
    url("img/cat@1x.avif") type("image/avif") 1x,
    url("img/cat@2x.jpeg") 2x,
    url("img/cat@1x.jpeg") 1x
);

  8. From GIF, JPG, PNG and AVIF - better use AVIF. With the same look, it`s 30% smaller.

  9. Always use max-width: 100%; and height: auto; on images, width dimension in html. Browser use html dimensions for calculating aspect-ration of images, for better performance.

  10. Use custom hotkey in vs code, emmet - update image size .

Squoosh API

Not maintainable

  1. Need install:

    • @squoosh/cli globally

  2. Change node to 16

  3. Commands:

                                    
squoosh-cli --mozjpeg '{quality: 75}' -d img/jpg *(*.jpg|*.png|*.avif|*.webp!(webp|avif|jpg))
squoosh-cli --avif '{quality: 100}' -d img/avif *(*.jpg|*.png|*.avif|*.webp!(webp|avif|jpg))

squoosh-cli --webp '{ quality: 90, lossless: 0, effort: 6 }' -d img/webp *(*.jpg|*.png|*.avif|*.webp!(webp|avif|jpg))

squoosh-cli --mozjpeg '{quality: 75}' -d jpg *(*.jpg|*.png|*.avif|*.webp!(webp|avif|jpg)) && squoosh-cli --avif '{quality: 100}' -d avif *(*.jpg|*.png|*.avif|*.webp!(webp|avif|jpg)) && squoosh-cli --webp '{ quality: 90, lossless: 0, effort: 6 }' -d webp *(*.jpg|*.png|*.avif|*.webp!(webp|avif|jpg))
