What is CSS?

CSS - Cascading Style Sheets

CSS describes the visual style and presentation of the content written in HTML

CSS consists of countless properties that developers use to format the content: properties about font, text, spacing, layout, etc. 

                        
h1 {                                                 |
    color: blue;          |                          |
    text-align: center;   | > Declaration block      | > A CSS rule
    font-size: 20px;      |                          |
}                                                    |

h1 - selector
font-size: 20px; - declaration/style
font-size - property
20px - value

Types

Inline CSS - inside the element with style attribute.
Internal CSS - write CSS code in style element which is inside head element.

separation of concerns - разделение интересов
External CSS - write CSS in external file .css and link it with link element.

Selectors

  1. Descendant selector

    • footer p {}

  2. Nested descendant selector

    • article footer p {}

  3. AND selector

    • .page.page--current {}

  4. ID selector

    • #author
    • Only 1 on the page
    • Нельзя выбратье сли начинается с цифры, но это легко обойти используя селектор по атрибуду [id="10"]

  5. Class selector

    • .author

  6. Прямой потомок, Direct children

    • .parent > .child {}

  7. Следующий соседний элемент, Adjacent sibling

    • .one + .item {}

  8. Все соседние элементы, General sibling

    • .item:nth-child(3) ~ .item {все соседние элементы .item после третьего}

  9. :not() selector

    • .grid:not(:last-child) {}

  10. :empty - обнаруживает пустые элементы, и задает им стили, а также можно скрыть при помощи display: none;

    • .item:empty {display:none}

  11. Селектор значения атрибута по подстроке, находящейся в его начале

    • ^ - карет/колпак
                                        
    <li data-type="todo-chore">Empty the bins</li>
    <li data-type="todo-exercise">Play football</li>

    [data-type^="todo"] {
      /* стили */
    }

  12. Селектор значения атрибута по имеющемуся в нем экземпляру подстроки

    • * - содержит
                                        
<p data-ingredients="scones cream jam">Will I get selected?</p>
Этот элемент можно выбрать следующим образом:

[data-ingredients*="cream"] {
  color: red;
}

  13. Селектор значения атрибута по подстроке, находящейся в его конце

    • $ - заканчивается подстрокой
                                        
<p data-ingredients="scones cream jam">Will I get selected?</p>
<p data-ingredients="toast jam butter">Will I get selected?</p>
<p data-ingredients="jam toast butter">Will I get selected?</p>

[data-ingredients$="jam"] {
    color: red;
}

  14. Attribute selector 

                                        
img:not([alt]),
img[alt=""] {
    border: 3px solid red;
}

[data-sausage] {
    /* стили */
}

Pseudo-classes

  1. li:first-child

    • Работает только если елемент буквально первый потомок. Лучший вариант использования, когда все елементы (потомки) одинаковые. Допустив как в списках - li.

  2. :only-child

    • Один и неповторимый потомок

  3. :only-of-type

    • Один и неповторимый потом по элементу

  4. li:nth-child(n/odd/even)

    n может использоваться следующими двумя способами:

    • в виде целого числа, например :nth-child(2), что приведет к выбору второго элемента
    • в виде числового выражения, например :nth-child(3n+1), благодаря чему выбор начнется с первого элемента и продолжится выбором каждого третьего элемента.

  5. nth-last-child(n), :nth-last-child(-n+3)

    • Выбирает третий элемент с конца, а после этого каждый элемент после него

  6. nth-of-type(n)

    • :nth-of-type(n + 4) - начиная с четвертого, каждый следующий
    • :nth-of-type(-n + 3) - начиная с третьего каждый предыдущий

  7. nth-last-of-type(n)

Box model

  • Content-box - default behavior

    1. Final element width: left-border + left-padding + width + right padding + right border

    2. Final element height: top-border + top-padding + height + bottom-padding + bottom-border

    3. With CSS: box-sizing: content-box;

  • Border-box

    1. Final element width: width

    2. Final element height: height

    3. With CSS: box-sizing: border-box;

Types of boxes

  • Block-level elements

    1. Elements are formatted visually as blocks

    2. Elements occupy 100% of parent element`s width, no matter the content

    3. Elements are stacked vertically by default, one after another

    4. The box-model applies as showed earlier

    5. Default elements: body, main, header, footer, section, nav, aside, div, h1-h6, p, ul, ol, li, etc.

    6. With CSS: display: block;

  • Inline elements

    1. Occupies only the space necessary for its content

    2. Causes no line-breaks after or before the element

    3. Box model applies in a different way: height and width do not apply

    4. Paddings and margins are applied only horizontally (left and right)

    5. Width and Height are not applies. Paddings and margins only horizontally

    6. With padding fill area inside elements expanded but not content around the box

    7. Default elements: a, img, strong, em, button, etc

    8. With CSS: display: inline;

  • Inline-block elements

    1. Images has inline-block behavior

    2. Look like inline from the outside, behaves like block-level on inside

    3. Occupies only content`s space | inline

    4. Causes no line-breaks | inline

    5. Box-model applies how it does in a block level boxes | block

    6. Default elements: img (it`s inline element, but doing like inline-block)

    7. With CSS: display: inline-block;

    8. Nice to use it with ::after and ::before pseudo-elements

Elements flow

  • Normal flow

    1. Default positioning: position: relative;

    2. Element is "in flow"

    3. Elements are simply laid out according to their order in the HTML code

  • Absolute Positioning

    1. Absolute positioning: position: absolute;

    2. Element is removed from the normal flow: "out of flow"

    3. No impact on surrounding elements, might overlap them

    4. We use top, bottom, left or right to offset the element from its relatively positioned container

    5. By default relation to viewport

Pseudo-elements

  1. Лучше использовать свойсво css display со значением inline-block для задавания отступов, ибо по стандарту это инлайн элемент.

  2. Позиционировать с помощью position:absolute, top, etc.

  3. With CSS: p::after, p::before, p::first-letter, p::first-line, ::selection, ::cue, ::marker по стандарту это инлайн элемент.

  4. p::after - last child element of p, need content="" property

  5. p::before - first child element of p, need content="" property
