CSS: Main

CSS (Cascading Style Sheets) is used to style and layout web pages — for example, to alter the font, color, size, and spacing of your content, split it into multiple columns, or add animations and other decorative features.

About

CSS - Cascading Style Sheets

CSS describes the visual style and presentation of the content written in HTML

CSS consists of countless properties that developers use to format the content: properties about font, text, spacing, layout, etc. 

                        
h1 {                                                 |
    color: blue;          |                          |
    text-align: center;   | > Declaration block      | > A CSS rule
    font-size: 20px;      |                          |
}                                                    |

h1 - selector
font-size: 20px; - declaration/style
font-size - property
20px - value

Inline CSS - inside the element with style attribute.

Box model

All elements are boxes.

  • Content-box - default behavior:

    1. Final element width: left-border + left-padding + width + right padding + right border

    2. Final element height: top-border + top-padding + height + bottom-padding + bottom-border

    3. With CSS: box-sizing: content-box;

  • Border-box:

    1. Final element width: width

    2. Final element height: height

    3. With CSS: box-sizing: border-box;

  • User agent stylesheets:

    1. Chromium

    2. Firefox

    3. Webkit

Types of boxes

  • Block-level elements:

    1. Elements are formatted visually as blocks

    2. Elements occupy 100% of parent element`s width, no matter the content

    3. Elements are stacked vertically by default, one after another

    4. The box-model applies as showed earlier

    5. Default elements: body, main, header, footer, section, nav, aside, div, h1-h6, p, ul, ol, li, etc.

    6. With CSS: display: block;

  • Inline elements:

    1. Occupies only the space necessary for its content

    2. Causes no line-breaks after or before the element

    3. Box model applies in a different way: height and width do not apply

    4. Paddings and margins are applied only horizontally (left and right)

    5. Width and Height are not applies. Paddings and margins only horizontally

    6. With padding fill area inside elements expanded but not content around the box

    7. Default elements: a, img, strong, em, button, etc

    8. With CSS: display: inline;

  • Inline-block elements:

    1. Images has inline-block behavior

    2. Look like inline from the outside, behaves like block-level on inside

    3. Occupies only content`s space | inline

    4. Causes no line-breaks | inline

    5. Box-model applies how it does in a block level boxes | block

    6. Default elements: img (it`s inline element, but doing like inline-block)

    7. With CSS: display: inline-block;

    8. Nice to use it with ::after and ::before pseudo-elements

Elements flow

  1. Normal flow

    • Default positioning: position: static;
    • Element is "in flow"
    • Elements are simply laid out according to their order in the HTML code

  2. Relative positioning

    • An element with position: relative will maintain its place in the document flow, even when you set these values. They will be relative to your element's position too.

  3. Absolute positioning

    • Absolute positioning: position: absolute;
    • Element is removed from the normal flow: "out of flow"
    • Will base the directional values from the relative parent's position.
    • No impact on surrounding elements, might overlap them
    • We use top, bottom, left or right to offset the element from its relatively positioned container
    • By default relation to viewport

  4. Fixed positioning

    • An element with position: fixed will base the directional values on the viewport.

  5. Sticky positioning

    • An element with position: sticky will only apply the directional values when it is in its docked/stuck state.

Cascade

MDN Cascade About cascade

The cascade algorithm is split into 4 distinct stages:

  1. Position and order of appearance: the order of which your CSS rules appear

  2. Specificity: an algorithm which determines which CSS selector has the strongest match

    • specifishity.com
    • specificity calculator
    • another specificity calculator

  3. Origin: the order of when CSS appears and where it comes from, whether that is a browser style, CSS from a browser extension, or your authored CSS

    • User agent base styles. These are the styles that your browser applies to HTML elements by default.
    • Local user styles. These can come from the operating system level, such as a base font size, or a preference of reduced motion. They can also come from browser extensions, such as a browser extension that allows a user to write their own custom CSS for a webpage.
    • Authored CSS. The CSS that you author.
    • Authored !important. Any !important that you add to your authored declarations.
    • Local user styles !important. Any !important that come from the operating system level, or browser extension level CSS.
    • User agent !important. Any !important that are defined in the default CSS, provided by the browser.

  4. Importance: some CSS rules are weighted more heavily than others, especially with the !important rule type

    Not all CSS rules are calculated the same as each other, or given the same specificity as each other.

    The order of importance, from least important, to most important is as follows:

    • normal rule type, such as font-size, background or color
    • animation rule type
    • !important rule type (following the same order as origin)
    • transition rule type

Inheritence

Inheritable properties

Inheritance control:

  1. The inherit keyword

    • Make any property inherit its parent's computed value with the inherit keyword. A useful way to use this keyword is to create exceptions. 
                                    
.my-component strong {
    font-weight: inherit;
}

  2. The initial keyword

    • Inheritance can cause problems with elements and initial provides a powerful reset option 
                                    
aside strong {
    font-weight: initial;
}

  3. The unset keyword

    • The unset property behaves differently if a property is inheritable or not. If a property is inheritable, the unset keyword will be the same as inherit. If the property is not inheritable, the unset keyword is equal to initial. 
                                    
/* Global color styles for paragraph in authored CSS */

p {
    margin-top: 2em;
    color: goldenrod;
}

/* The p needs to be reset in asides, so you can use unset */

aside p {
    margin: unset;
    color: unset;
}

Colors

  • Color keywords:

    1. 148 named colors in CSS

    2. transparent is a fully transparent color. It is also the initial value of background-color

    3. currentColor is the contextual computed dynamic value of the color property. If you have a text color of red and then set the border-color to be currentColor, it will also be red. If the element that you define currentColor on doesn't have a value for color defined, currentColor will be computed by the cascade instead

Sizing units

  • Numbers can also be used in the following places:

    1. When setting values for filters: filter: sepia(0.5) applies a 50% sepia filter to the element

    2. When setting opacity: opacity: 0.5 applies a 50% opacity

    3. In color channels: rgb(50, 50, 50)

    4. To transform an element: transform: scale(1.2) scales the element by 120% of its initial size.

  • Percentages:

    1. Set margin or padding as a percentage, they will be a portion of the parent element's width, regardless of direction.

    2. Set a transform value as a percentage, it is based on the element with the transform set.

  • Absolute lenghths:

    1. All absolute lengths resolve against the same base, making them predictable wherever they're used in your CSS.

    2. Absolute leghts

  • Relative lengths:

    1. A relative length is calculated against a base value, much like a percentage. The difference between these and percentages is that you can contextually size elements. This means that CSS has units such as ch that use the text size as a basis, and vw which is based on the width of the viewport (your browser window). Relative lengths are particularly useful on the web due to its responsive nature.

    2. Fonts size relative units

    3. Viewport-relative units

  • Miscellaneous units:

    1. There are some other units which have been specified to deal with particular types of values.

    2. Angle units

    3. Resolution units

Pseudo-classes

MDN Pseudo-classes WebDev Pseudo-classes

  1. li:last-child

    li:first-child

    • Only works later if the element is literally the first. The best use case is when all elements (children) are tightened. Assuming as in lists - li.

  2. li:nth-child(n/odd/even)

    n can be used in the following two ways:

    • as an integer, for example :nth-child(2), which will result in the selection of the second element
    • as a numeric expression, e.g. :nth-child(3n+1), so that the selection starts with the first element and continues with every third element.

    • Selects the third element from the end, and after that then every element after that
